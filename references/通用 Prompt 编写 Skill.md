# 通用 Prompt 编写 Skill

## 目标

帮助用户把任何领域的需求整理成高质量 Prompt。

这个 Skill 适用于写作、编程、设计、研究、运营、商业分析、学习辅导、产品规划、法律资料整理、数据分析、视频脚本、图片生成、自动化任务等场景。

核心目标是让 AI 明确知道：

1. 要完成什么
2. 基于什么信息完成
3. 按什么步骤完成
4. 使用什么工具或资料
5. 达到什么质量标准
6. 输出成什么格式
7. 哪些事情不能做
8. 如何验证结果是否可用

## 核心原则

好的 Prompt 不是简单命令，它应该是一份任务说明书。

编写 Prompt 时，应尽量包含以下信息：

1. 任务目标
2. 背景上下文
3. 输入材料
4. 角色定位
5. 执行流程
6. 质量标准
7. 输出格式
8. 限制条件
9. 验证方式
10. 迭代规则

## 通用 Prompt 结构

推荐使用下面这个结构编写任何领域的 Prompt。

```text
你将扮演：
{角色}

我要完成的任务是：
{任务目标}

背景信息：
{背景上下文}

输入材料：
{文件、链接、数据、图片、文本、已有草稿、参考案例}

请按照以下流程执行：
1. 先理解任务目标和背景。
2. 再分析输入材料。
3. 找出关键信息、问题、机会或约束。
4. 制定执行方案。
5. 按方案完成最终输出。
6. 自查结果是否符合要求。
7. 如果发现不确定信息，请明确标注。

输出要求：
{格式、字数、语言风格、表格、Markdown、JSON、代码、文章、报告等}

质量标准：
{准确、清晰、可执行、符合目标用户、符合业务目标、逻辑严谨、可复用等}

限制条件：
{不能做什么、不能出现什么、必须避免什么、风格禁忌、安全边界}

最终请输出：
1. 核心结果
2. 关键依据
3. 可执行步骤
4. 风险或注意事项
5. 后续优化建议
```

## Prompt 编写流程

### 第一步：把模糊需求改成明确目标

用户经常会说：

```text
帮我写个方案
```

需要改成：

```text
请帮我写一份面向中小企业老板的 AI 自动化落地方案，目标是让他们理解如何用 AI 降低人工成本。要求内容通俗，步骤具体，可以直接用于公众号文章。
```

目标要包含：

1. 给谁看
2. 解决什么问题
3. 用在什么场景
4. 希望产生什么结果

## 第二步：补充上下文

没有上下文，AI 很容易泛泛输出。

上下文可以包括：

1. 项目背景
2. 用户画像
3. 当前阶段
4. 已有材料
5. 过往经验
6. 竞品案例
7. 业务目标
8. 风格偏好
9. 技术栈
10. 交付场景

模板：

```text
背景信息：
这个任务用于 {使用场景}。
目标用户是 {用户群体}。
他们当前的问题是 {具体痛点}。
我希望最终结果能帮助他们 {预期收益}。
已有材料包括 {输入材料}。
```

## 第三步：指定角色

角色不要只写“你是专家”，要和任务目标绑定。

普通写法：

```text
你是一个写作专家。
```

更好的写法：

```text
你是一个擅长把复杂 AI 技术讲给普通读者的公众号作者。你需要用口语化、清晰、具体的表达，把技术概念转化成普通人能理解的内容。
```

角色要包含：

1. 专业领域
2. 服务对象
3. 工作方式
4. 输出风格
5. 判断标准

角色模板：

```text
你是一个 {领域} 专家，擅长 {能力}。你的目标用户是 {用户群体}。你需要用 {风格} 的方式，帮助他们完成 {任务结果}。
```

## 第四步：给出执行流程

不要只写“帮我完成”，要写“先做什么，再做什么”。

通用流程模板：

```text
请按照以下步骤执行：

1. 先理解我的目标和背景。
2. 提炼任务中的关键信息。
3. 找出影响结果的核心变量。
4. 给出执行思路。
5. 生成最终内容。
6. 自查是否符合我的要求。
7. 如果有不确定内容，请单独列出。
```

如果是分析类任务：

```text
请按照以下步骤执行：

1. 先总结材料的核心观点。
2. 再拆解它的逻辑结构。
3. 找出最有价值的方法。
4. 分析它适合哪些场景。
5. 提炼成可复用的方法论。
6. 最后整理成可直接执行的清单。
```

如果是创作类任务：

```text
请按照以下步骤执行：

1. 先确定目标受众。
2. 再确定内容主题和情绪。
3. 设计内容结构。
4. 写出第一版内容。
5. 检查语言是否自然。
6. 优化标题、开头和结尾。
7. 输出可直接发布的版本。
```

如果是编程类任务：

```text
请按照以下步骤执行：

1. 先阅读相关文件和调用链。
2. 找出问题根因。
3. 提出 2 到 3 个方案。
4. 等我确认方案后再修改代码。
5. 修改后运行测试。
6. 总结修改内容、验证结果和风险点。
```

如果是设计类任务：

```text
请按照以下步骤执行：

1. 先理解品牌定位和使用场景。
2. 再确定视觉风格。
3. 拆解画面元素。
4. 生成设计方案。
5. 检查画面是否统一、清晰、可用。
6. 输出最终提示词或设计说明。
```

## 第五步：指定输出格式

输出格式越明确，结果越稳定。

常见输出格式：

1. Markdown 文档
2. 表格
3. JSON
4. 分步骤清单
5. 文章成稿
6. 视频脚本
7. 图片提示词
8. 产品需求文档
9. 代码文件
10. 会议纪要
11. SOP 流程
12. 检查清单

Markdown 模板：

```text
请用 Markdown 输出，结构如下：

# 标题

## 一、核心结论

## 二、背景分析

## 三、执行步骤

## 四、注意事项

## 五、可复用模板
```

表格模板：

```text
请用表格输出，字段包括：

1. 模块
2. 作用
3. 具体做法
4. 注意事项
5. 示例
```

JSON 模板：

```text
请输出 JSON，字段包括：

{
  "task_goal": "",
  "context": "",
  "steps": [],
  "output_format": "",
  "constraints": [],
  "quality_check": []
}
```

## 第六步：加入质量标准

质量标准决定 AI 如何判断“好不好”。

可以这样写：

```text
质量标准：

1. 内容必须具体，避免空泛。
2. 每一步都要可执行。
3. 语言要符合目标用户理解能力。
4. 不要堆概念。
5. 重要结论要给出理由。
6. 如果信息不足，请标注假设。
7. 最终结果要可以直接使用。
```

不同领域可以替换质量标准。

写作类：

```text
要求语言口语化，有真实感，少用套话。开头要抓人，结尾要有记忆点。
```

研究类：

```text
要求区分事实、推断和观点。重要结论需要说明依据。
```

编程类：

```text
要求先理解现有代码，再修改。修改后必须说明测试结果和潜在风险。
```

设计类：

```text
要求画面主体明确，风格统一，信息层级清晰，避免元素堆砌。
```

商业类：

```text
要求建议可落地，有优先级，有成本意识，有风险判断。
```

## 第七步：加入限制条件

限制条件可以减少跑偏。

模板：

```text
限制条件：

1. 不要输出泛泛的建议。
2. 不要编造不存在的信息。
3. 不确定的内容要标注。
4. 不要改变原始事实。
5. 不要省略关键步骤。
6. 不要使用过度正式的表达。
7. 不要输出无法执行的空话。
```

也可以加入风格限制：

```text
语言风格：

1. 简洁
2. 口语化
3. 具体
4. 有判断
5. 有人味
6. 适合直接发布
```

## 第八步：加入验证方式

好的 Prompt 要让 AI 自查。

通用自查模板：

```text
输出完成后，请根据以下标准自查：

1. 是否完成了任务目标
2. 是否使用了我提供的上下文
3. 是否存在泛泛而谈
4. 是否每一步都能执行
5. 是否符合输出格式
6. 是否有不确定内容
7. 是否需要补充风险提示
```

编程验证：

```text
请说明你运行了哪些测试、结果如何、还有哪些没有验证。
```

写作验证：

```text
请检查标题是否有吸引力，开头是否能留住读者，正文是否顺畅，结尾是否有记忆点。
```

设计验证：

```text
请检查主体是否清晰、画面是否干净、风格是否统一、是否适合实际使用。
```

## 第九步：加入迭代规则

复杂任务不要指望一次完成。

模板：

```text
如果第一次输出不够好，请按照以下方式迭代：

1. 先指出当前结果的问题。
2. 再说明优化方向。
3. 最后给出优化后的完整版本。
```

也可以要求多版本：

```text
请先给出 3 个不同方向的方案，每个方案说明优点、缺点和适合场景。最后推荐一个最适合当前目标的方案。
```

## 通用 Prompt 生成模板

当用户需要为任何任务编写 Prompt 时，使用下面模板。

```text
请根据我的需求，帮我生成一个高质量 Prompt。

我的需求：
{用户原始需求}

请你先分析这个需求属于什么类型，然后补全一个可执行 Prompt。

生成的 Prompt 必须包含：

1. 角色定位
2. 任务目标
3. 背景上下文
4. 输入材料
5. 执行流程
6. 输出格式
7. 质量标准
8. 限制条件
9. 验证方式
10. 迭代规则

要求：
1. Prompt 要清晰、具体、可执行。
2. 不要只写一句简单命令。
3. 要让 AI 知道先做什么、后做什么。
4. 要让 AI 知道什么结果才算合格。
5. 如果用户信息不足，请合理补充默认假设，并标注出来。
```

## 万能 Prompt 框架

```text
你是一个 {角色}。

任务目标：
{我要完成的事情}

背景信息：
{为什么要做、给谁用、当前情况是什么}

输入材料：
{文件、文本、数据、截图、链接、参考案例}

执行流程：
1. 先理解任务目标。
2. 分析输入材料。
3. 提炼关键信息。
4. 找出主要问题或机会。
5. 设计解决方案。
6. 生成最终结果。
7. 按质量标准自查。
8. 标注不确定信息和风险点。

输出格式：
{指定格式}

质量标准：
1. 结果必须具体。
2. 逻辑必须清晰。
3. 步骤必须可执行。
4. 语言必须符合目标用户。
5. 不确定内容必须标注。
6. 最终结果必须能直接使用。

限制条件：
{不能出现的内容、不能做的事情、风格限制、安全边界}

验证方式：
请在最后检查：
1. 是否满足任务目标
2. 是否覆盖全部要求
3. 是否存在遗漏
4. 是否存在不确定信息
5. 是否可以直接使用

最终输出：
{最终交付物}
```

## Prompt 优化方法

当用户提供一个旧 Prompt 时，按以下方式优化。

```text
请帮我优化下面这个 Prompt。

原 Prompt：
{原始内容}

请按照以下标准优化：
1. 目标是否明确
2. 上下文是否充分
3. 执行步骤是否清楚
4. 输出格式是否明确
5. 质量标准是否具体
6. 限制条件是否完整
7. 是否方便 AI 执行
8. 是否容易得到稳定结果

请输出：
1. 原 Prompt 的问题
2. 优化思路
3. 优化后的 Prompt
4. 使用建议
```

## Prompt 质量评分表

每次生成 Prompt 后，可以用下面标准评分。

| 指标 | 说明 | 分数 |
|---|---|---|
| 目标明确 | 是否清楚说明要完成什么 | 1 到 5 |
| 上下文完整 | 是否提供背景、对象、材料 | 1 到 5 |
| 流程清晰 | 是否说明先做什么后做什么 | 1 到 5 |
| 输出明确 | 是否指定格式和交付物 | 1 到 5 |
| 标准具体 | 是否说明什么结果算好 | 1 到 5 |
| 限制充分 | 是否说明不能做什么 | 1 到 5 |
| 可执行性 | AI 是否能直接按步骤完成 | 1 到 5 |
| 可复用性 | 是否能迁移到类似任务 | 1 到 5 |

评分低于 32 分时，需要继续优化。

## 使用这个 Skill 时的默认行为

当用户说“帮我写一个 Prompt”时，应先判断任务类型，再自动补全结构。

当用户信息不足时，可以先给出一个带默认假设的版本，并标注假设。

当用户要求“直接给我可用的”时，不要长篇解释，直接输出完整 Prompt。

当用户要求“做成 Skill”时，应输出包含目标、适用场景、执行流程、模板、质量标准和示例的 Markdown 文档。

当用户要求“优化 Prompt”时，应先指出原 Prompt 的问题，再给出优化后的完整版本。

## 核心口诀

一个好 Prompt，至少要说清楚：

谁来做  
做什么  
为什么做  
基于什么做  
怎么做  
做到什么程度  
输出成什么样  
哪些不能做  
怎么检查好坏  
下一轮怎么改
